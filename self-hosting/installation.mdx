## Overview

This guide walks you through installing and running both the Aegis bot and dashboard on your own server. Make sure you have completed the [Prerequisites](/self-hosting/prerequisites) guide before continuing.

## Step 1 — Clone the Repositories

Clone both the bot and dashboard repositories:

```bash
git clone https://github.com/your-org/bot.git
git clone https://github.com/your-org/dashboard.git
```

## Step 2 — Set Up the Bot

Navigate to the bot directory:

```bash
cd bot
```

Install dependencies:

```bash
bun install
```

Create the environment file:

```bash
cp .env.example .env
```

Open `.env` and fill in your values:

```env
# Discord
DISCORD_TOKEN=your_bot_token_here
DISCORD_CLIENT_ID=your_client_id_here
DISCORD_CLIENT_SECRET=your_client_secret_here

# Database
DATABASE_URL=postgresql://aegisuser:your_password@localhost:5432/aegis

# YouTube
YOUTUBE_API_KEY=your_youtube_api_key_here

# Dashboard
BOT_API_SECRET=generate_a_random_secret_here
```

Push the database schema:

```bash
bun run db:push
```

Start the bot:

```bash
bun run start
```

<Info>
  For development, use `bun run dev` instead — this enables hot reloading.
</Info>

## Step 3 — Set Up the Dashboard

Open a new terminal and navigate to the dashboard directory:

```bash
cd dashboard
```

Install dependencies:

```bash
bun install
```

Create the environment file:

```bash
cp .env.example .env
```

Open `.env` and fill in your values:

```env
# Discord OAuth2
DISCORD_CLIENT_ID=your_client_id_here
DISCORD_CLIENT_SECRET=your_client_secret_here

# Database
DATABASE_URL=postgresql://aegisuser:your_password@localhost:5432/aegis

# NextAuth
NEXTAUTH_SECRET=generate_a_random_secret_here
NEXTAUTH_URL=http://localhost:3000

# Bot API
BOT_API_URL=http://localhost:4000
BOT_API_SECRET=same_secret_as_bot_env
```

Start the dashboard:

```bash
bun run start
```

The dashboard will be available at [http://localhost:3000](http://localhost:3000).

## Step 4 — Invite Your Bot

Generate an invite link for your bot using the Discord Developer Portal:

1. Go to **OAuth2 → URL Generator**
2. Select the **bot** and **applications.commands** scopes
3. Select the **Administrator** permission
4. Copy and open the generated URL
5. Select your server and click **Authorize**

## Running in Production

For production, we recommend using a process manager like **PM2** to keep Aegis running in the background.

Install PM2:

```bash
npm install -g pm2
```

Start the bot with PM2:

```bash
cd bot
pm2 start "bun run start" --name aegis-bot
```

Start the dashboard with PM2:

```bash
cd dashboard
pm2 start "bun run start" --name aegis-dashboard
```

Save the PM2 process list so it restarts on reboot:

```bash
pm2 save
pm2 startup
```

## Keeping Aegis Updated

To update Aegis to the latest version:

```bash
cd bot
git pull
bun install
bun run db:push
pm2 restart aegis-bot

cd ../dashboard
git pull
bun install
pm2 restart aegis-dashboard
```

<Warning>
  Always read the changelog before updating — some updates may include breaking changes or require database migrations.
</Warning>

## Troubleshooting

### Bot is not responding

- Make sure your `DISCORD_TOKEN` is correct and hasn't been reset
- Check that the bot has the correct permissions in your server
- Check the bot logs with `pm2 logs aegis-bot`

### Database connection failed

- Make sure PostgreSQL is running: `sudo systemctl status postgresql`
- Double check your `DATABASE_URL` in the `.env` file
- Make sure the database user has the correct permissions

### Dashboard won't load

- Make sure `NEXTAUTH_URL` matches the URL you're accessing the dashboard from
- Check that your Discord OAuth2 redirect URI matches your dashboard URL
- Check the dashboard logs with `pm2 logs aegis-dashboard`
